Soulmate email system

All Supabase templates in one place.

This folder now contains the full Soulmate email template bundle for Supabase authentication emails and security notification emails. Open any file directly, then paste its HTML into the matching Supabase dashboard template slot.

Authentication Emails

Confirm sign up

Used when a new user must verify their email before account activation.

Dashboard slot: Confirm sign up
Key variable: {{ .ConfirmationURL }}

Magic link

Used for passwordless sign-in links.

Dashboard slot: Magic link
Key variable: {{ .ConfirmationURL }}

Reset password

Used when a user requests a password reset email.

Dashboard slot: Reset password
Key variable: {{ .ConfirmationURL }}

Invite user

Used when an invite-based account needs to be accepted.

Dashboard slot: Invite user
Key variable: {{ .ConfirmationURL }}

Change email address

Used to confirm a requested email-address update.

Dashboard slot: Change email address
Key variables: {{ .ConfirmationURL }}, {{ .NewEmail }}

Reauthentication

Used when the user must confirm a sensitive action with a short-lived code or link.

Dashboard slot: Reauthentication
Key variables: {{ .Token }}, {{ .ConfirmationURL }}
Security Notification Emails

Password changed

Confirmation that an account password has already been updated.

Project setting: Password changed notification
Key variable: {{ .Email }}

Email address changed

Confirms a completed email-address update.

Project setting: Email address changed notification
Key variables: {{ .OldEmail }}, {{ .Email }}

Phone number changed

Confirms a completed phone-number update.

Project setting: Phone number changed notification
Key variables: {{ .OldPhone }}, {{ .Phone }}

Identity linked

Confirms that a new sign-in provider was attached to the account.

Project setting: Identity linked notification
Key variable: {{ .Provider }}

Identity unlinked

Confirms that a sign-in provider was removed from the account.

Project setting: Identity unlinked notification
Key variable: {{ .Provider }}

MFA method added

Confirms that a new MFA factor was enrolled.

Project setting: MFA method added notification
Key variable: {{ .FactorType }}

MFA method removed

Confirms that an MFA factor was removed.

Project setting: MFA method removed notification
Key variable: {{ .FactorType }}
